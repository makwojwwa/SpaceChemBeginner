Programming mindset

SpaceChem applies a visual programming language, which means that you don't have to write code to create algorithms. All commands (called instructions A building block for algorithms in reactors. Must be placed inside the reactor along the path of a waldo so that it can be executed. in-game) are in the form of symbols, which are placed on the grid in a desired sequence. Thanks to that, the game is relatively easy to grasp even for a person without any programming knowledge and experience.

Bear in mind that you will not learn how to program by playing SpaceChem. However, by solving successive levels you have an opportunity to acquire a programming mindset, which might help you become a programmer in the future, if you wish so.

What you can learn

  • Every solution is based on inputs and outputs.

    The input is what you receive from outside (for example, data entered by a website user while logging in) and the output is what your program produces based on the input and the instructions you add.

  • You can get different inputs, which require different solutions.

    You have to process the input such that the required output is obtained. That is why if you get two different atoms in the same input area, you have to recognize them and create separate algorithms for them so that they can be processed in the required manner. When using the real life example mentioned above, your program used by the website has to recognize the first name entered by the user and know what to do with this particular piece of information so that the user can log in. If a different first name is entered, your program must log the user in a different account.

  • You might require more than one input to produce required output.

    Sometimes you need two different atoms coming from two different input areas to produce a required molecule. In other words, you need a username and a password entered by a user in two different fields so that the user can be logged in by your program. You won't achieve your goal if you don't get either of them or if you accidentally apply an algorithm that matches a different input than actually present.

  • Tasks are complex but can be divided into sections.

    A task of creating a molecule (e.g. logging a user in) is complex: it consists of several sections. Handle one section, test if it does what it is supposed to and then proceed to another section. Try to combine all sections such that the required output is created.

  • Tasks can be similar and can be re-used.

    If you know how to bind an oxygen atom with a carbon atom (for example, a username with a password), you can use this template to create an algorithm for binding an hydrogen atom with a nitrogen atom (for example, a different username with a different password).

  • Programs are made of loops.

    You get two atoms and you have to create a molecule. You recognize them, apply the sequence of instructions, produce the output molecule and move back to the beginning. Your program is ready to get new inputs and create new outputs.

  • Programs can contain branches.

    Sometimes your inputs contain atoms that you don't need or that appear in a wrong sequence – this happens in production assignments. In this case, you can create branches and direct those wrong atoms straight to the recycler. The same can happen in our real-life example: if your program expects a particular input after the first one (a password after a username), it must be prepared for a situation where, for instance, a username is entered again instead of a password. A programmer must expect the unexpected and prepare branches to handle such incorrect inputs.

  • Downstream reactors must wait for input from upstream reactors.

    In production assignments, the last reactor must receive inputs from downstream reactors to execute the final reaction and produce the final output. It can happen that one input is received much earlier than the other, so the reactor cannot proceed with its algorithm. This situation can be compared to logging in a bank's website with user authentication. Let's say that the user entered the username and the password but a security code is still required. The program will not log the user in until the security code is entered.
